# Contributing to DM0 Projects

Thank you for your interest in contributing to DM0 Technology Corp projects.

## How to Contribute

### Reporting Issues

- Check existing issues first to avoid duplicates
- Use the issue templates when available
- Include enough detail to reproduce bugs

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Ensure tests pass
5. Commit with clear messages
6. Push and open a PR against `main`

### Commit Messages

Write clear, concise commit messages:

```
Short summary (50 chars or less)

Longer explanation if needed. Wrap at 72 characters.
Explain what and why, not how.
```

### Code Style

Follow the conventions established in each repository:

- **Unison**: See project-specific style guidelines
- **Rust**: Run `cargo fmt` and `cargo clippy`
- **Python**: Follow PEP 8, use `black` for formatting
- **Haskell**: Run `ormolu` or `fourmolu`
